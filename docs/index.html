ObserverZ.com Documentation

Complete reference guide for the tag-driven content discovery platform

Welcome to ObserverZ.com

ObserverZ.com is a modern, production-ready web application designed for Gen Z users who want to discover trending topics, build personal journals, and earn rewards by sharing their interests.

What is ObserverZ?

ObserverZ is a two-engine platform:

  • Data Engine: Continuously fetches, parses, and analyzes content from multiple sources
  • Presentation Engine: Renders dynamic index pages for each topic with beautiful UI

Key Capabilities

🔥 Trending Topics

Real-time trending feed with 72-hour rolling windows

📚 Personal Journals

Create and manage custom topic collections

💰 Monetization

Earn $1 per 1000 views on shared content

🌍 Global Analytics

Geographic insights by country and region

Quick Start Guide

Get ObserverZ.com running in 5 minutes

Docker Compose (Recommended)

The fastest way to get started is with Docker Compose.

Step 1: Clone Repository

git clone https://github.com/yourusername/observerz.git
cd observerz

Step 2: Configure Environment

cp .env.example .env
nano .env  # Edit with your configuration

Step 3: Start Services

docker-compose up -d

Step 4: Initialize Database

docker-compose exec backend pnpm db:push

Step 5: Access Application

  • Frontend: http://localhost:3001
  • Backend API: http://localhost:3000
  • Nginx: https://localhost
✅ Success! Your application is now running. You can start exploring and customizing.

Local Development Setup

Install and run locally for development

Prerequisites

  • Node.js 20+
  • pnpm 8+
  • PostgreSQL 16+
  • Redis 7+
  • Git

Installation Steps

# 1. Clone repository
git clone https://github.com/yourusername/observerz.git
cd observerz

# 2. Install dependencies
pnpm install

# 3. Create database
createdb observerz
createuser observerz
psql -c "ALTER USER observerz WITH PASSWORD 'password';"

# 4. Configure environment
cp .env.example .env
nano .env

# 5. Run migrations
pnpm db:push

# 6. Start development servers
# Terminal 1:
pnpm dev

# Terminal 2:
cd client && pnpm dev

Access Application

  • Frontend: http://localhost:5173
  • Backend: http://localhost:3000
💡 Note: For local development, you can use Docker for PostgreSQL and Redis instead of installing them locally.

Understanding the ObserverZ system design

Architecture Overview

┌─────────────────────────────────────────────────────────────┐

└────────────────────────┬────────────────────────────────────┘

        ┌────────────────┼────────────────┐
        │                │                │
        ▼                ▼                ▼
   ┌─────────┐    ┌──────────┐    ┌──────────┐
   │PostgreSQL│    │  Redis   │    │ Workers  │
   │Database  │    │  Cache   │    │ (Celery) │
   └─────────┘    └──────────┘    └──────────┘

Data Flow

  1. Ingestion: URLs submitted via admin interface or email
  2. Fetching: HTTP client fetches content
  3. Parsing: Boilerplate removal and extraction
  4. Tagging: NLP-based keyword extraction
  5. Ranking: Popularity scoring
  6. Storage: Data persisted to PostgreSQL
  7. Delivery: API serves cached results

Component Responsibilities

Component Responsibility
Frontend User interface, state management, real-time updates
Backend API Business logic, authentication, data validation
Database Persistent data storage, indexing, transactions
Cache Query result caching, session management
Workers Background jobs, data ingestion, report generation

Database Schema

Complete database structure and relationships

Core Tables

  • users: User accounts and profiles
  • sources: Content sources (news sites, blogs)
  • articles: Individual articles and content
  • tags: Topic tags and keywords
  • article_tags: Many-to-many relationship

Analytics Tables

  • click_events: Track user clicks
  • geo_rollups: Geographic analytics
  • user_ranks: User reputation scores
  • api_usage: API call tracking for billing

User Feature Tables

  • journals: User's personal collections
  • walls: User's shareable walls
  • stars: User's favorite articles
  • pins: User's pinned articles
  • comments: User comments
  • filters: Content filters

Monetization Tables

  • wallets: User OCT balances
  • wallet_tx: Transaction history
  • subscriptions: Active subscriptions
  • payouts: Monthly earnings
💡 Note: See SCHEMA.md for detailed table definitions and relationships.

API Reference

Complete API endpoint documentation

Authentication

All protected endpoints require a valid JWT token:

Authorization: Bearer YOUR_JWT_TOKEN

Core Endpoints

Tags

  • GET /api/trpc/tags.trending - Get trending topics
  • GET /api/trpc/tags.bySlug - Get tag by slug
  • GET /api/trpc/tags.feed - Get articles for a tag
  • GET /api/trpc/tags.related - Get related tags

Articles

  • GET /api/trpc/articles.byId - Get article by ID
  • GET /api/trpc/articles.search - Search articles
  • POST /api/trpc/articles.ingest - Ingest new article
  • GET /api/trpc/articles.geo - Get geographic analytics

Journals

  • POST /api/trpc/journals.create - Create journal
  • GET /api/trpc/journals.list - List user journals
  • GET /api/trpc/journals.byId - Get public journal

Wallet

  • GET /api/trpc/wallet.balance - Get wallet balance
  • GET /api/trpc/wallet.transactions - Get transaction history
  • POST /api/trpc/wallet.buyOCT - Purchase OCT tokens

Database Configuration

Configure and optimize your database

PostgreSQL Setup

# Create database and user
sudo -u postgres psql

CREATE USER observerz WITH PASSWORD 'secure_password';
CREATE DATABASE observerz OWNER observerz;
GRANT ALL PRIVILEGES ON DATABASE observerz TO observerz;
\q

Run Migrations

# Using Docker
docker-compose exec backend pnpm db:push

# Locally
pnpm db:push

Database Optimization

# Analyze query performance
EXPLAIN ANALYZE SELECT * FROM articles WHERE created_at > NOW() - INTERVAL '7 days';

# Create indices
CREATE INDEX idx_articles_created_at ON articles(created_at DESC);
CREATE INDEX idx_article_tags_tag_id ON article_tags(tag_id);

# Vacuum and analyze
VACUUM ANALYZE;

Backup and Restore

# Backup
docker-compose exec postgres pg_dump -U observerz observerz > backup.sql

# Restore
docker-compose exec -T postgres psql -U observerz observerz < backup.sql
⚠️ Important: Always backup your database before major operations.

Troubleshooting

Common issues and solutions

Database Connection Failed

# Check PostgreSQL is running
docker-compose ps postgres

# Check connection string
echo $DATABASE_URL

# Test connection
docker-compose exec postgres psql -U observerz -d observerz -c "SELECT 1;"

# View PostgreSQL logs
docker-compose logs postgres

Redis Connection Failed

# Check Redis is running
docker-compose ps redis

# Test Redis connection
docker-compose exec redis redis-cli ping

# View Redis logs
docker-compose logs redis

Frontend Not Loading

# Check frontend service
docker-compose ps frontend

# Check frontend logs
docker-compose logs frontend

# Verify API connectivity
curl http://localhost:3000/health

# Clear browser cache and reload

High Memory Usage

# Check memory usage
docker stats

# Reduce Redis memory
docker-compose exec redis redis-cli CONFIG SET maxmemory 512mb

# Optimize PostgreSQL
docker-compose exec postgres psql -U observerz -c "VACUUM ANALYZE;"

Slow API Responses

# Check database query performance
docker-compose exec postgres psql -U observerz -c "SELECT * FROM pg_stat_statements ORDER BY mean_time DESC LIMIT 10;"

# Check Redis hit rate
docker-compose exec redis redis-cli INFO stats

# Review slow query log
docker-compose logs backend | grep "slow"
💡 Tip: Check logs first when troubleshooting. Most issues are documented in the logs.

Backup Strategy

#!/bin/bash
BACKUP_DIR="/backups/observerz"
DATE=$(date +%Y%m%d_%H%M%S)

# Backup database
docker-compose exec -T postgres pg_dump -U observerz observerz | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Backup Redis
docker-compose exec -T redis redis-cli BGSAVE

# Keep only last 7 days
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +7 -delete
